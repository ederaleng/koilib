Interface Entries

These entries definitions are used to serialize and deserialize contract operations. Each entry contains a name (defined in the key field), id (entry point id), inputs (abi describing the serialization of the inputs), and outputs (abi describing the serialization of the outputs). See Abi.

example
const entries = {
  transfer: {
    id: 0x62efa292,
    inputs: {
      type: [
        {
          name: "from",
          type: "string",
        },
        {
          name: "to",
          type: "string",
        },
        {
          name: "value",
          type: "uint64",
        },
      ],
    },
  },
  balance_of: {
    id: 0x15619248,
    inputs: { type: "string" },
    outputs: { type: "uint64" },
  },
};
